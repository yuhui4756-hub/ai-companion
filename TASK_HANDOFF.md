# 任务交接规则

所有专业线程完成任务后，使用下面格式写交接，方便总控决定下一步交给谁。

## 标准交接格式

```md
## 交接记录

- 交接来源：
- 建议交接给：
- 当前阶段：
- 完成内容：
- 修改文件：
- 验证方式：
- 遗留问题：
- 需要用户参与：
- 下一步建议：
```

## 交接判断

| 当前问题 | 交给谁 |
| --- | --- |
| 不知道该做什么功能 | 产品经理 |
| 聊天冷冰冰、不像伴侣 | AI人设设计 |
| 不知道系统怎么拆 | 技术架构 |
| 需要写代码或修 bug | 开发实现 |
| 不确定是否达标 | 测试验收 |
| 文档散乱、上下文丢失 | 记录交接 |
| 需要用户登录、授权、确认风险 | 总控 |

## 总控对用户提问规则

总控只在以下情况向用户提问：

- 需要登录、验证码、授权或付费确认。
- 需要用户选择明显影响方向的产品定位。
- 涉及删除数据、上传隐私、公开发布、付费接口调用等高风险操作。
- 本地环境或第三方平台需要用户亲自操作。

其他专业问题默认由对应线程自行查证、判断并推进。
