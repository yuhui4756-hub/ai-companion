# AI伴侣

一个中文 BYOK（Bring Your Own Key，自带 API Key）AI 伴侣网页 Demo。第一版支持伴侣选择、OpenAI 兼容接口配置、聊天和本地长期记忆管理。

## 已实现功能

- 5 类伴侣预设：温柔朋友、轻恋爱陪伴、理性支持、治愈陪伴、角色扮演。
- BYOK 配置：`providerName`、`baseURL`、`model`、`apiKey`。
- OpenAI Chat Completions 兼容接口调用，当前为非流式。
- 长期记忆本地管理：手动新增、查看、编辑、删除。
- 聊天时会轻量自动识别明确偏好、称呼、禁忌和情绪模式，并沉淀为可管理的本地记忆。
- 聊天时会按相关性把长期记忆注入提示词。
- 清楚的错误提示：缺少配置、认证失败、网络失败、模型错误、响应格式异常。
- 预留 `channel-adapter`，第一版只实现 Web，QQ/微信为占位。

## 小白启动说明

先确认电脑已经安装 Node.js。然后在项目目录运行：

```bash
npm install
npm run dev
```

看到类似下面的地址后，用浏览器打开：

```text
http://127.0.0.1:5173/
```

停止运行：回到运行 `npm run dev` 的命令行窗口，按 `Ctrl + C`。

## 怎么使用

1. 打开网页后，先在左侧或聊天页选择伴侣类型。
2. 进入“设置”，填写你的 OpenAI 兼容接口：
   - `baseURL` 示例：`https://api.openai.com/v1`
   - `model` 示例：填写你的服务商支持的模型名
   - `apiKey`：填写你自己的 Key
3. 回到“聊天”开始对话。
4. 进入“记忆”手动添加、编辑或删除长期记忆。

## 隐私说明

第一版没有后端服务器。API Key、聊天记录和长期记忆默认保存在当前浏览器的 `localStorage`，项目不内置、不提交任何真实 Key。

注意：浏览器会直接请求你填写的接口。如果某些服务商不允许网页跨域访问，可能会出现网络失败，这种情况后续需要加本地代理。
