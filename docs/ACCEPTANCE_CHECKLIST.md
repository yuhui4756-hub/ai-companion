# 验收清单

## 工程化升级阶段新增验收

- 本地代理：前端模型请求经过本地代理或 Electron 主进程代理，不再只依赖浏览器直连供应商接口。
- 配置安全：测试日志、控制台、导出文件和错误提示中不出现真实 API Key、token、Cookie 或扫码凭证。
- 数据迁移：旧 `localStorage` 中的伴侣、长期记忆、风格摘要和聊天数据有明确迁移或备份恢复路径。
- SQLite/本地库：桌面版核心数据可以稳定读写；重启应用后数据仍存在。
- RAG/知识库：用户导入一段文本后，聊天能检索并注入相关片段；删除资料后不再注入。
- 隐私边界：知识库内容只在本地保存；注入给模型服务商的内容必须来自当前用户明确导入或聊天上下文。
- 兼容体验：恋爱伴侣创建、聊天、记忆管理、风格摘要和设置入口不能因为工程化改造退化。
- 构建验证：至少通过 `npx tsc --noEmit` 和 `npm run build`。

## 产品体验

- 用户第一次打开能理解这是 AI 伴侣，而不是普通机器人。
- 用户能选择不同伴侣类型。
- 默认回复有温度，不像客服模板。
- 用户问实际问题时，AI 能给出可执行建议。
- 用户切换角色扮演风格后，AI 风格有明显变化。

## 长期记忆

- 系统能保存用户明确表达的重要偏好。
- 记忆能在后续聊天中被合理使用。
- 用户可以查看全部记忆。
- 用户可以编辑记忆。
- 用户可以删除记忆。
- 删除后的记忆不再进入后续提示词。

## API 接入

- 用户可以填写 API Key。
- 用户可以配置 baseURL 和 model。
- API Key 错误时有可理解的错误提示。
- 网络失败时有可理解的错误提示。
- 代码仓库不包含真实 API Key。

## 安全边界

- AI 不冒充医生、心理咨询师、律师等专业身份。
- 高风险自伤内容有安全回应。
- 不鼓励用户完全依赖 AI 替代现实关系。
- 不主动索要敏感隐私。

## 工程质量

- 项目可以按 README 启动。
- 主要模块职责清楚。
- 没有明显重复代码。
- UI 在常见桌面和手机宽度下不重叠、不溢出。
