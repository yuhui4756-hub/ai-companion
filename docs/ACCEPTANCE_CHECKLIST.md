# 验收清单

## 产品体验

- 用户第一次打开能理解这是 AI 伴侣，而不是普通机器人。
- 用户能选择不同伴侣类型。
- 默认回复有温度，不像客服模板。
- 用户问实际问题时，AI 能给出可执行建议。
- 用户切换角色扮演风格后，AI 风格有明显变化。

## 长期记忆

- 系统能保存用户明确表达的重要偏好。
- 记忆能在后续聊天中被合理使用。
- 用户可以查看全部记忆。
- 用户可以编辑记忆。
- 用户可以删除记忆。
- 删除后的记忆不再进入后续提示词。

## API 接入

- 用户可以填写 API Key。
- 用户可以配置 baseURL 和 model。
- API Key 错误时有可理解的错误提示。
- 网络失败时有可理解的错误提示。
- 代码仓库不包含真实 API Key。

## 安全边界

- AI 不冒充医生、心理咨询师、律师等专业身份。
- 高风险自伤内容有安全回应。
- 不鼓励用户完全依赖 AI 替代现实关系。
- 不主动索要敏感隐私。

## 工程质量

- 项目可以按 README 启动。
- 主要模块职责清楚。
- 没有明显重复代码。
- UI 在常见桌面和手机宽度下不重叠、不溢出。
