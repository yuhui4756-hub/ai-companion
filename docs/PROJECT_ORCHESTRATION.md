## 当前阶段目标

- 让项目从“本地优先的 AI 伴侣产品原型”升级成更适合作为 AI 应用开发简历项目的工程化版本。
- 第一优先级：引入本地后端代理，解决浏览器直连模型接口的跨域、密钥暴露和供应商适配问题。
- 第二轮优先方向：引入 Python/FastAPI 本地服务作为可展示的后端工程模块，承接本地代理、SQLite 持久化、迁移脚本和最小 RAG 检索中的至少一个可验收闭环。
- 第二优先级：把核心数据从纯 `localStorage` 迁移到更可靠的本地持久化层，优先评估 SQLite。
- 第三优先级：加入最小可用的本地知识库/RAG 能力，让伴侣能引用用户明确导入的资料，而不是只靠聊天记忆。

## 约束

- 继续坚持本地优先，默认不上传用户 API Key、聊天记录、长期记忆或导入资料。
- 不引入账号系统、云同步、角色市场、真实 QQ/微信接入作为本阶段目标。
- 不要求用户做专业技术判断；专业取舍由对应线程给出建议，总控拍板。
- 当前阶段先完成需求、架构、实现和验收闭环；发布安装包或公开 release 需要用户另行确认。
