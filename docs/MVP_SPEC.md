# MVP 产品规格

## 第一版目标

做出一个可运行的中文网页 Demo，让用户可以：

- 选择或轻量自定义一个 AI 伴侣。
- 填写自己的 API Key。
- 和伴侣持续聊天。
- 让伴侣提供情绪价值和基础问题解答。
- 查看、编辑、删除长期记忆。

## 用户流程

1. 用户打开网页。
2. 用户选择伴侣类型和默认聊天风格。
3. 用户填写 API 服务商、模型、API Key 和接口地址。
4. 用户进入聊天界面。
5. 系统根据伴侣人设、用户画像、长期记忆组织提示词。
6. 用户聊天过程中，系统自动识别可沉淀的长期记忆。
7. 用户可进入记忆管理页查看、编辑、删除记忆。

## 伴侣类型

- 温柔朋友：稳定倾听、鼓励、日常陪伴。
- 轻恋爱陪伴：更亲密，但避免过度成人化。
- 理性支持：帮用户拆问题、做计划、给建议。
- 治愈陪伴：语气柔和，适合压力、低落、焦虑时使用。
- 角色扮演：允许动漫感或设定感，但仍遵守安全边界。

## MVP 功能优先级

### P0 必须有

- 单页网页聊天界面。
- 伴侣类型选择。
- 用户自填 API Key。
- OpenAI 兼容接口调用。
- 本地长期记忆存储。
- 记忆查看、编辑、删除。
- 基础错误提示，如 API Key 错误、网络失败、模型返回失败。

### P1 重要但可后置

- 多供应商预设，如 OpenAI、DeepSeek、通义千问、Kimi、智谱等。
- 伴侣头像和名称自定义。
- 用户首次问卷。
- 聊天记录搜索。
- 导入/导出配置。

### P2 后续增强

- QQ/微信入口适配。
- 多设备同步。
- 云端账号系统。
- 伴侣市场或角色模板分享。
- 语音输入输出。

## 非目标

- 第一版不承诺真实接入 QQ/微信。
- 第一版不做医疗、心理治疗或法律等高风险专业服务。
- 第一版不保存或代管用户商业 API Key 到远程服务器。
